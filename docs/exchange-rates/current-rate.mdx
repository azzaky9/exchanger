# Get Current Rate

<Badge text="API" variant="note" />
<Badge text="Active Only" variant="success" />

Returns the **markup-applied exchange rates** currently configured by the admin. Only responds
successfully when the exchange rate is marked as **active**. If no active rate exists, the
endpoint returns `404` Рђћ use [`/reference-rate`](/docs/exchange-rates/reference-rate) instead.

---

## Endpoint

- **Method:** `GET`
- **Path:** `/api/exchange-rates/current`

---

## Authentication

```http
Authorization: users API-Key YOUR_API_KEY
```

| Role    | Access |
| ------- | ------ |
| `admin` | РюЁ     |
| `user`  | РюЁ     |
| Guest   | РЮї     |

---

## Response Fields

| Field             | Type     | Description                                                             |
| ----------------- | -------- | ----------------------------------------------------------------------- |
| `phpToUsdtRate`   | `number` | Platform rate: how much USDT 1 PHP buys, after markup deduction.        |
| `usdtToPhpRate`   | `number` | Platform rate: how much PHP 1 USDT gets, after markup deduction.        |
| `updatedAt`       | `string` | ISO 8601 timestamp when the rate was last saved.                        |

---

## Success Response `200`

```json
{
  "phpToUsdtRate": 0.000016824,
  "usdtToPhpRate": 57.33,
  "updatedAt": "2026-04-17T07:30:00.000Z"
}
```

---

## Error Responses

| Status | Message                        | Reason                                                         |
| ------ | ------------------------------ | -------------------------------------------------------------- |
| `401`  | `Unauthorized`                 | Missing or invalid API key / session.                          |
| `404`  | `No active exchange rate found`| The rate record has `isActive: false` or no record exists.     |
| `500`  | `<error string>`               | Unexpected server error.                                       |

:::caution
When the admin disables the exchange rate (`isActive: false`), this endpoint returns **404**.
Your client app must handle this gracefully Рђћ fall back to [`/reference-rate`](/docs/exchange-rates/reference-rate)
for display and allow the user to continue transacting.
:::

## TypeScript / Fetch Example

```ts
interface CurrentRateResponse {
  phpToUsdtRate: number
  usdtToPhpRate: number
  updatedAt: string
}

async function getCurrentRate(apiKey: string): Promise<CurrentRateResponse | null> {
  const res = await fetch('/api/exchange-rates/current', {
    method: 'GET',
    headers: {
      Authorization: `users API-Key ${apiKey}`,
    },
  })

  // Rate is inactive Рђћ fall back to reference rate
  if (res.status === 404) return null

  const data = await res.json()
  if (!res.ok) throw new Error(data?.error ?? 'Failed to fetch current rate')

  return data as CurrentRateResponse
}
```
